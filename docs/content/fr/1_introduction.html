Introduction à Veridot

Veridot est un protocole de vérification de jetons cryptographiques à haute performance et une bibliothèque Java de niveau entreprise conçue pour les architectures distribuées à grande échelle. Il résout le compromis classique entre la validation de jetons à latence ultra-faible et la révocation immédiate.

Le Paradoxe de la Révocation Distribuée

Les architectures modernes de microservices exigent une authentification rapide, sécurisée et hautement scalable. Traditionnellement, les développeurs doivent choisir entre deux modèles architecturaux imparfaits :

Le Compromis de l'Authentification

1. L'Introspection d'État (OAuth2 / Requêtes de Base de Données)
Le service de vérification effectue une requête réseau synchrone vers un serveur d'autorisation central ou une base de données pour chaque requête API. Bien que cela permette une révocation immédiate, cela crée un goulot d'étranglement de performance, ajoute 10-150ms de latence par requête, et introduit un point de défaillance unique (SPOF).

2. La Validation de JWT sans état (Stateless)
Les services de vérification valident cryptographiquement les jetons localement à l'aide d'une clé publique mise en cache. C'est extrêmement rapide (inférieur à la milliseconde) et résistant aux pannes réseau. Cependant, une fois le jeton signé, il ne peut plus être révoqué avant l'expiration de sa durée de vie (TTL), exposant le système à des failles si le jeton est volé.

La Solution Veridot

Veridot résout ce paradoxe. Il offre la rapidité d'une vérification locale sans état couplée à la réactivité instantanée d'un système de révocation. Il repose sur une architecture de réplication des métadonnées :

  • Cache Cryptographique Local : Les nœuds de vérification maintiennent un cache local et thread-safe des clés de session actives, des clés de rotation publiques et des contraintes de configuration.
  • Réplication Asynchrone de l'État : Les transitions d'état (nouvelles sessions, révocations, changements de règles) sont publiées dans un journal distribué à haut débit (comme Apache Kafka ou une base de données SQL) et propagées de manière asynchrone à tous les vérificateurs.
  • Invariants de Filigrane (Version Watermark) : Un protocole mathématique strict de suivi de version protège le système contre les attaques par rejeu, les partitions réseau et le rollback d'anciens états.

Scénarios Réels Résolus par Veridot

Scénario A : Vol d'une Clé d'API

Un attaquant vole une clé d'API utilisateur. Avec un JWT classique, la clé reste valide jusqu'à son expiration (ex. 1 heure). Avec Veridot, l'administrateur émet un événement de révocation. En quelques millisecondes, la révocation est propagée aux passerelles API et le jeton volé est rejeté.

Scénario B : Panne de la Base de Données Centrale

Dans un système d'introspection d'état, si la base de données s'arrête, tout le trafic API s'arrête. Sous Veridot, les validateurs continuent à valider les jetons existants grâce à leur cache local, assurant la continuité d'activité pendant la résolution de la panne.
