API & Configuration Reference

Complete documentation for Veridot's core classes, public interfaces, and operational configuration keys.

Core Public Interfaces

The library's surface is split into clean, modular interfaces depending on the target role:

1. DataSigner

Used by issuing services to generate signed session tokens.

public interface DataSigner {
    String sign(Object data, Configurer configurer);
}

2. TokenVerifier

Used by verifying services/gateways to validate tokens locally.

public interface TokenVerifier {
    <T> VerifiedData<T> verify(String token, Function<String, T> deserializer)
        throws BrokerExtractionException;
}

3. TokenRevoker

Used by administration tools or security services to invalidate tokens.

public interface TokenRevoker {
    void revoke(Scope scope, String sequenceId);
}

4. TokenTracker

Used to query the active state and details of active or revoked sessions.

public interface TokenTracker {
    List<SessionInfo> listActive(Scope scope);
}

Configuration Reference

Configure Veridot instances at startup using the following variables (or environment properties):

Property / Var Default Description
VDOT_KEYS_ROTATION_MINUTES 60 Interval at which the ephemeral keys are rotated.
VDOT_RECONCILIATION_INTERVAL_MINUTES 10 Reconciliation scheduler loop frequency.
VDOT_CAPABILITY_CACHE_TTL_SECONDS 60 TTL for positive authorization capabilities cache.
VDOT_CAPABILITY_NEGATIVE_CACHE_TTL_SECONDS 5 TTL for negative/failed authorization capabilities cache.
