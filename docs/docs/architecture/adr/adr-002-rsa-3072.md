# ADR-002: RSA-3072 Ephemeral Key Size

* **Status**: Accepted
* **Date**: 2026-06-27

## Context

Veridot supports RSA-based signature algorithms for environments where Ed25519 or ECDSA are not permitted by corporate security policies. In V2, the implicit ephemeral RSA key size was 2048 bits. However, NIST recommendations state that 2048-bit RSA keys do not provide adequate security margins post-2030.

## Decision

We standardize on **RSA-3072** as the minimum allowed key length for all ephemeral RSA keys generated by the library:
- `Config.MIN_RSA_KEY_LENGTH` is set to `3072`.
- Any attempt to generate or verify an envelope with a shorter RSA key will trigger validation errors.

## Consequences

- Improved security margins conforming to post-2030 standards.
- Marginally higher CPU usage during ephemeral key generation and verification compared to RSA-2048 (mitigated by using Ed25519 as the primary recommendation).
