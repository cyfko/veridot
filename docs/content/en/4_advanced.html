Advanced Implementation Guides

Dive deeper into the operational features of Veridot V4, including custom metadata storage, real-time revocation, and key rotation strategies.

Token Revocation

Veridot provides two levels of revocation: single session revocation and group-wide revocation. Revocation is performed using the TokenRevoker interface.

No Physical Deletions

In Veridot V4, revocation does not delete configuration or keys from the database. Instead, it writes a signed LIVENESS envelope with status set to REVOKED. Verifying nodes immediately detect this signal and fail-fast.

// Revoke a single active session (using its unique sequenceId)
service.revoke(Scope.group("user-1234"), "session-xyz-123");

// Revoke all sessions within a group (e.g., logging a user out of all devices)
service.revoke(Scope.group("user-1234"), null);

Distributed Metadata Brokers

Brokers are responsible for propagating state changes. Veridot includes pre-built broker adapters for distributed backends:

1. Apache Kafka Broker (Recommended for high scale)

Provides sub-second, real-time status replication across regions using an append-only transaction log. In veridot-kafka:

Properties kafkaProps = new Properties();
kafkaProps.put("bootstrap.servers", "localhost:9092");
kafkaProps.put("embedded.db.path", "/var/lib/veridot-rocksdb");
Broker kafkaBroker = new KafkaBroker(kafkaProps);

2. Relational Database Broker (SQL)

Uses relational databases like PostgreSQL, MySQL, or Microsoft SQL Server as the metadata broker. Relies on polling and triggers. In veridot-databases:

HikariDataSource dataSource = new HikariDataSource();
dataSource.setJdbcUrl("jdbc:postgresql://localhost:5432/veridot");
Broker sqlBroker = new DatabaseBroker(dataSource, "broker_messages");

Ephemeral Key Rotation

To reduce key exposure risk, Veridot signers perform automatic key rotation. The KeyRotationService generates short-lived ephemeral key pairs (RSA or Ed25519) in the background. The corresponding public keys are published as KEY_EPOCH envelopes to the broker so that verifiers can dynamically fetch and cache them for signature verification.
