Changelog: Veridot V4

This page lists the release notes and major improvements introduced in Veridot V4.0.0.

v4.0.0 (2026-06-28)

Veridot V4.0.0 is a major release introducing the V4 wire protocol, structural capabilities, and enhanced fencing features.

New Features

  • Structural Capability Delegation (§6): Introduces the CAPABILITY entry type. Publishing rights on scopes are now validated cryptographically against an explicit delegation chain terminating at the TrustRoot.
  • Monotonic Capacity Fencing (§9): Adds the FENCE entry type carrying a monotonic counter. This guarantees transaction serializeability for concurrent session quota writes without synchronous distributed locks.
  • Off-heap RocksDB Cache: The Kafka broker module now includes an embedded RocksDB database to support caching millions of active sessions with a minimal JVM heap footprint.
  • Zero-copy Envelope Parsing: Redesigned wire format parser enables sub-microsecond envelope validation.

Changes & Enhancements

  • Envelope Magic identifier changed to 0x56 0x44 (Protocol Version 4).
  • Enhanced five-minute clock skew NTP validation guard for KEY_EPOCH and LIVENESS entries.
  • Renamed ReconciliationService to WatermarkReconciler.

Deprecated & Removed

  • Removed the legacy V3 flat-file key resolver configuration.
  • Removed synchronous Jwks polling fallback.
