Guide de Démarrage Rapide

Ce guide vous aidera à installer Veridot Core, configurer une instance de vérification, signer un jeton de session cryptographique, le valider et exécuter une boucle de révocation.

Étape 1 : Installation

Ajoutez la bibliothèque Veridot Core à votre fichier de build :

<dependency>
    <groupId>io.github.cyfko</groupId>
    <artifactId>veridot-core</artifactId>
    <version>4.0.0</version>
</dependency>
implementation 'io.github.cyfko:veridot-core:4.0.0'

Étape 2 : Signature & Vérification d'un Jeton

Créez une classe nommée VeridotDemo.java dans votre projet. Ce code illustre le cycle de vie complet d'une session sous le protocole Veridot :

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import io.github.cyfko.veridot.core.exceptions.*;
import java.security.*;

public class VeridotDemo {
    public static void main(String[] args) throws Exception {
        // 1. Initialise un InMemoryBroker pour les tests locaux
        Broker broker = new InMemoryBroker();
        
        // 2. Génère une paire de clés cryptographiques à long terme (RSA 2048) pour le signataire
        KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
        keyGen.initialize(2048);
        KeyPair keyPair = keyGen.generateKeyPair();
        
        // 3. Définit le TrustRoot pour résoudre la clé publique dynamiquement
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override
            public PublicKey resolve(String issuer) {
                // En production, résolvez cette clé depuis un KMS ou un coffre-fort de clés
                return keyPair.getPublic();
            }
            @Override
            public boolean isRootIdentity(String issuer) {
                // Identifie l'identité émettrice racine
                return "auth-authority-01".equals(issuer);
            }
        };

        // 4. Instancie le service GenericSignerVerifier
        GenericSignerVerifier service = new GenericSignerVerifier(
            broker,
            trustRoot,
            "auth-authority-01",
            keyPair.getPrivate(),
            (byte) 0x01 // Algorithme de signature de l'enveloppe (SHA256withRSA)
        );

        System.out.println("Service Veridot initialisé avec succès.");

        // 5. Signe une session pour "user-99" avec une validité de 10 minutes
        Configurer configurer = BasicConfigurer.builder()
            .groupId("user-99")
            .sequenceId("session-abc-456")
            .validity(600) // Durée en secondes (10 minutes)
            .distribution(DistributionMode.DIRECT) // Renvoie un jeton JWT standard
            .build();

        String token = service.sign("{\"role\":\"admin\",\"name\":\"Alice\"}", configurer);
        System.out.println("\nJeton JWT généré :\n" + token);

        // 6. Vérifie le jeton immédiatement
        System.out.println("\n--- Vérification du jeton actif ---");
        try {
            VerifiedData<String> verified = service.verify(token, payload -> payload);
            System.out.println("Vérification réussie !");
            System.out.println("Données de la payload : " + verified.data());
            System.out.println("La session est active : " + verified.isLive());
        } catch (Exception e) {
            System.err.println("Échec de la vérification : " + e.getMessage());
        }

        // 7. Révocation manuelle de la session
        System.out.println("\n--- Révocation de la session ---");
        service.revoke(Scope.group("user-99"), "session-abc-456");

        // 8. Tente une nouvelle vérification. Elle doit échouer instantanément
        System.out.println("\n--- Vérification du jeton révoqué ---");
        try {
            service.verify(token, payload -> payload);
            System.err.println("ERREUR CRITIQUE : Le jeton a été vérifié après révocation !");
        } catch (BrokerExtractionException e) {
            System.out.println("Succès : La vérification du jeton a échoué comme prévu. Raison : " + e.getMessage());
        }

        // Fermeture des planificateurs de réconciliation en arrière-plan
        service.close();
    }
}
Version de Java requise

Assurez-vous que votre projet utilise Java 17 ou supérieur, car Veridot exploite intensivement les fonctionnalités modernes du langage (records, pattern matching sur switch).
