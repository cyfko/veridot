Introduction à Veridot

Veridot est un protocole de vérification de jetons cryptographiques à haute performance et une bibliothèque Java conçue pour les architectures distribuées à grande échelle. Il résout le compromis historique entre la validation locale sans état (JWT) et la révocation immédiate avec état.

Points Clés

Pourquoi choisir Veridot V4 ?
  • ✓ Latence inférieure à la milliseconde : La validation cryptographique s'effectue entièrement en mémoire via des caches locaux thread-safe, évitant tout appel réseau synchrone.
  • ✓ Révocation Décentralisée : Les événements de révocation se propagent de manière asynchrone aux microservices en moins d'une seconde via un broker de réplication.
  • ✓ Garantie Anti-Rollback : Veridot V4 applique un filigrane de version monotone sur tous les messages d'état, empêchant les attaques par rejeu.
  • ✓ Prêt pour la Production : Intégrations natives pour Spring Boot, Apache Kafka, et les bases de données relationnelles.

Le Compromis de l'Authentification

Les architectures modernes nécessitent une authentification rapide et sécurisée. Les approches classiques imposent un choix cornélien :

Métrique Opérationnelle JWT Classique (Sans État) Introspection OAuth2 (Avec État) Veridot V4
Latence de Vérification < 1 ms (Cryptographique locale) 15 ms - 150 ms (Appel réseau synchrone) < 1 ms (Vérification en cache local)
Délai de Révocation Impossible (Attente de la fin du TTL) Instantané < 1 seconde (Propagation asynchrone)
Charge Base de Données Nulle Élevée (Requêtes pour chaque appel) Très Faible (Lecture seule des mises à jour)
Résilience aux Pannes Réseau Excellente (Vérification autonome) Très Mauvaise (Panne si le serveur d'auth est down) Élevée (Maintien du cache et des watermarks)

Exemple d'Utilisation Minimal

L'exemple suivant montre l'initialisation du service, la signature d'un jeton, sa validation locale, puis sa révocation immédiate :

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import java.security.KeyPairGenerator;
import java.security.KeyPair;
import java.security.PublicKey;

public class ExempleMinimal {
    public static void main(String[] args) throws Exception {
        // Initialisation d'un broker en mémoire pour les tests
        Broker broker = new InMemoryBroker();
        
        // Génération d'une paire de clés
        KeyPair keyPair = KeyPairGenerator.getInstance("RSA").generateKeyPair();
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override public PublicKey resolve(String issuer) { return keyPair.getPublic(); }
            @Override public boolean isRootIdentity(String issuer) { return "auth-issuer".equals(issuer); }
        };

        // Initialisation de Veridot
        GenericSignerVerifier veridot = new GenericSignerVerifier(
            broker, trustRoot, "auth-issuer", keyPair.getPrivate(), (byte) 0x01
        );

        // Signature du jeton
        Configurer configurer = BasicConfigurer.builder()
            .groupId("user-123")
            .sequenceId("session-xyz")
            .validity(3600)
            .build();
        String token = veridot.sign("{\"user\":\"Alice\"}", configurer);

        // Vérification locale
        VerifiedData verified = veridot.verify(token, payload -> payload);
        System.out.println("La session est-elle active ? " + verified.isLive());

        // Révocation instantanée
        veridot.revoke(Scope.group("user-123"), "session-xyz");
        veridot.close();
    }
}
