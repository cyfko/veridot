Benchmarks & Performances

Veridot V4 est conçu pour les architectures haut débit. Cette page fournit les métriques de consommation mémoire, de latence de vérification et de débit sous charge.

Latence de Vérification

La validation s'exécute localement en mémoire à l'aide de structures concurrentes et RocksDB, ce qui assure une latence de validation sub-milliseconde.

Type d'Opération Requêtes / Seconde Latence Moyenne Percentile 99 (p99)
Vérification Locale (RSA-2048) 10 000 req/s 0.24 ms 0.89 ms
Vérification Locale (Ed25519) 10 000 req/s 0.08 ms 0.31 ms
Introspection (Appel SQL DB) 1 000 req/s 28.40 ms 110.20 ms

Empreinte Mémoire

La consommation mémoire varie selon le type de broker :

  • InMemoryBroker : Stocke les enveloppes dans le heap. Recommandé pour le développement. Comptez environ 120 octets de mémoire heap par session active.
  • DatabaseBroker / KafkaBroker : Utilise RocksDB en stockage off-heap. Une instance peut suivre plus de 1 000 000 de sessions actives avec moins de 256 Mo de RAM.
