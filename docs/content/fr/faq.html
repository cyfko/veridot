Foire Aux Questions

Cette FAQ regroupe les questions fréquentes et les conseils de dépannage pour Veridot.

Pourquoi mon jeton est-il rejeté ?

Si la signature cryptographique est valide mais que la vérification échoue, vérifiez :

  • Expiration du statut actif : Le heartbeat (liveness) a expiré. Assurez-vous que l'émetteur publie régulièrement l'attestation LIVENESS(ACTIVE) avant le timestamp validUntil.
  • Dérive d'horloge : Si l'écart d'heure entre verifiers et signers dépasse 5 minutes, la validation de KEY_EPOCH échouera. Activez NTP.
  • Version obsolète : L'enveloppe reçue possède une version inférieure ou égale au filigrane (watermark) local.

Pourquoi les révocations mettent-elles du temps à se propager ?

Si une révocation met plus d'une seconde à se propager :

  • Partitions Kafka : Vérifiez la répartition de vos partitions et assurez-vous que chaque validateur utilise un groupe de consommateurs unique pour éviter les blocages.
  • Indexation SQL : Dans le cas d'un broker SQL, vérifiez que l'index sur la colonne version est bien actif.

La bibliothèque Veridot est-elle thread-safe ?

Oui. La classe GenericSignerVerifier ainsi que les adaptateurs de brokers sont thread-safe et conçus pour être déclarés comme Singletons dans des environnements multi-threadés.

Puis-je valider des jetons sur mobile ou navigateur ?

Non. Les applications mobiles et les navigateurs ne peuvent pas stocker de clés de confiance de manière sécurisée ni se connecter au broker. Ils doivent appeler une passerelle API intermédiaire qui valide le jeton via le SDK Veridot.
