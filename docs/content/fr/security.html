Sécurité Opérationnelle & Directives

L'exploitation de Veridot en production exige le respect rigoureux des contraintes de synchronisation d'horloge et de gestion des clés.

Synchronisation d'Horloge (NTP)

Veridot s'appuie sur des horodatages précis pour définir la validité des clés et des sessions. Une dérive d'horloge peut introduire des failles de sécurité ou provoquer des rejets inattendus.

Seuil Maximal de Dérive Temporelle

L'écart de temps maximal toléré entre un nœud d'émission et un validateur est de 5 minutes. Tous les serveurs doivent synchroniser leurs horloges système via NTP.

Exemple de Configuration NTP (systemd-timesyncd)

Éditez le fichier de configuration /etc/systemd/timesyncd.conf sur vos instances Linux :

[Time]
NTP=pool.ntp.org time.google.com
FallbackNTP=ubuntu.pool.ntp.org

Activez et vérifiez l'état de la synchronisation à l'aide de la console :

sudo systemctl restart systemd-timesyncd
timedatectl status

Invariants TrustRoot & KMS

L'interface TrustRoot constitue la racine de confiance pour toutes les validations cryptographiques. Suivez ces invariants :

  • Conception Fail-Closed (Verrouillage par défaut) : Si le TrustRoot ne parvient pas à contacter le KMS pour résoudre une clé publique, il doit immédiatement lever une exception et rejeter le jeton.
  • Moindre Privilège : Les instances de validation n'ont besoin que d'un accès en lecture au broker. Les clés privées de signature doivent rester confinées dans le module HSM/KMS.
  • Rotation des Clés : Renouvelez les clés racines annuellement. Les clés éphémères d'époque doivent être régénérées toutes les heures (paramètre VDOT_KEYS_ROTATION_MINUTES).
