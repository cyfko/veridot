API & Configuration Reference

This reference guide details Veridot's core interfaces, exceptions, configuration keys, and environment variables.

Java Public API Surfaces

1. DataSigner

The interface used to issue and sign new user session tokens.

public interface DataSigner {
    /**
     * Generates a signed cryptographic token for the given payload.
     *
     * @param data The payload object to serialize and sign.
     * @param configurer The configuration options (validity, sequenceId, distribution mode).
     * @return The signed JWT (DIRECT mode) or the opaque message ID (INDIRECT mode).
     * @throws DataSerializationException If the payload cannot be serialized.
     */
    String sign(Object data, Configurer configurer) throws DataSerializationException;
}

2. TokenVerifier

The interface used to validate and extract payloads from tokens locally.

public interface TokenVerifier {
    /**
     * Validates the signature, liveness, and configuration of the token.
     *
     * @param token The JWT or message ID to verify.
     * @param deserializer Function to deserialize the JSON payload to type T.
     * @return A VerifiedData container containing the payload and validity info.
     * @throws BrokerExtractionException If the metadata cannot be resolved or is stale.
     */
    <T> VerifiedData<T> verify(String token, Function<String, T> deserializer)
        throws BrokerExtractionException;
}

3. TokenRevoker

The interface used to revoke active sessions cryptographically.

public interface TokenRevoker {
    /**
     * Revokes a specific session sequence or all sessions within a group.
     *
     * @param scope The scope of the target (e.g. Scope.group("user-99")).
     * @param sequenceId The unique session key to revoke, or null to revoke the entire group.
     */
    void revoke(Scope scope, String sequenceId);
}

Detailed Configuration Properties

Configure the Veridot engine using environment variables or Java system properties at startup:

Property Name Env Variable Default Type Description
veridot.keys.rotation.minutes VDOT_KEYS_ROTATION_MINUTES 60 integer Frequency at which ephemeral signing key pairs are rotated in the background to mitigate key exposure.
veridot.reconciliation.interval.minutes VDOT_RECONCILIATION_INTERVAL_MINUTES 10 integer Time interval between periodic watermark reconciliations. Triggers broker scans to update local baseline version marks.
veridot.capability.cache.ttl.seconds VDOT_CAPABILITY_CACHE_TTL_SECONDS 60 integer Duration for caching successfully verified publisher configurations and authorization rights.
veridot.capability.negative.cache.ttl.seconds VDOT_CAPABILITY_NEGATIVE_CACHE_TTL_SECONDS 5 integer Duration for caching failed capability verifications. Protects underlying storage from brute-force lookup attacks.
veridot.max.sessions.default VDOT_MAX_SESSIONS_DEFAULT -1 integer Fallback limit for the maximum number of concurrent active sessions per group scope. -1 means unlimited.
