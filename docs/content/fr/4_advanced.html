Guides d'Implémentation Avancés

Cette section détaille comment intégrer Veridot dans des architectures de production réelles, configurer les caches et déployer des brokers distribués.

Broker Base de Données en Production (SQL)

L'utilisation d'une base de données relationnelle (PostgreSQL, MySQL, SQL Server) comme broker est idéale pour les architectures microservices classiques. Cela nécessite un pool de connexions (comme HikariCP) et une table dédiée. Voici une configuration prête pour la production :

import com.zaxxer.hikari.HikariConfig;
import com.zaxxer.hikari.HikariDataSource;
import io.github.cyfko.veridot.core.Broker;
import io.github.cyfko.veridot.databases.DatabaseBroker;

public class DatabaseBrokerFactory {
    public static Broker createPostgresBroker() {
        HikariConfig config = new HikariConfig();
        config.setJdbcUrl("jdbc:postgresql://db-server.internal:5432/veridot_db");
        config.setUsername("veridot_app");
        config.setPassword("secure-database-password");
        
        // Paramètres d'optimisation de performance
        config.setMaximumPoolSize(15);
        config.setMinimumIdle(5);
        config.setIdleTimeout(30000);
        config.setConnectionTimeout(5000);
        config.addDataSourceProperty("cachePrepStmts", "true");
        config.addDataSourceProperty("prepStmtCacheSize", "250");
        config.addDataSourceProperty("prepStmtCacheSqlLimit", "2048");

        HikariDataSource dataSource = new HikariDataSource(config);
        
        // "broker_messages" est la table de stockage des enveloppes binaires
        return new DatabaseBroker(dataSource, "broker_messages");
    }
}

Schéma de la base de données (Exemple DDL PostgreSQL)

CREATE TABLE broker_messages (
    storage_key BYTEA PRIMARY KEY,
    entry_bytes BYTEA NOT NULL,
    version BIGINT NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_broker_messages_version ON broker_messages(version);

Broker Apache Kafka en Production

Pour les déploiements à très grande échelle et multi-régions, Apache Kafka est le broker de métadonnées recommandé. Il fournit une réplication des événements ultra-rapide. Le KafkaBroker utilise une instance **RocksDB** embarquée pour les lectures locales ultra-rapides :

import io.github.cyfko.veridot.core.Broker;
import io.github.cyfko.veridot.kafka.KafkaBroker;
import java.util.Properties;

public class KafkaBrokerFactory {
    public static Broker createKafkaBroker() {
        Properties props = new Properties();
        
        // Serveurs du cluster Kafka
        props.put("bootstrap.servers", "kafka-broker-01:9092,kafka-broker-02:9092");
        
        // Répertoire local de cache persistant RocksDB
        props.put("embedded.db.path", "/var/lib/veridot/rocksdb_cache");
        
        // Topic Kafka utilisé pour la réplication (doit être pré-créé)
        props.put("broker.topic", "veridot-state-replication");
        
        // Configuration de sécurité SASL_SSL
        props.put("security.protocol", "SASL_SSL");
        props.put("sasl.mechanism", "PLAIN");
        props.put("sasl.jaas.config", "org.apache.kafka.common.security.plain.PlainLoginModule required username=\"app\" password=\"secret\";");

        return new KafkaBroker(props);
    }
}

Protection des Bases de Données via le Cache Négatif

Dans les architectures distribuées, les passerelles API sont régulièrement la cible de requêtes non autorisées. Si chaque échec d'autorisation déclenchait une requête SQL synchrone pour vérifier la capacité de l'émetteur, la base de données serait rapidement submergée (vulnérabilité aux attaques par déni de service).

Rôle du Cache Négatif

Lorsqu'un validateur traite un jeton signé par un émetteur non autorisé, l'autorisation échoue. Veridot met cet échec en cache localement pendant une courte durée (définie par VDOT_CAPABILITY_NEGATIVE_CACHE_TTL_SECONDS, par défaut 5s). Durant cette fenêtre, toutes les requêtes ultérieures de cet émetteur sont immédiatement rejetées en local sans interroger le broker, protégeant ainsi votre base de données des attaques par force brute à haut débit.
