Référence API & Configuration

Documentation complète des classes fondamentales, des interfaces publiques et des clés de configuration opérationnelles de Veridot.

Interfaces Publiques Majeures

L'interface de programmation est segmentée en rôles distincts pour plus de modularité :

1. DataSigner

Utilisé par les services d'émission pour générer les jetons signés.

public interface DataSigner {
    String sign(Object data, Configurer configurer);
}

2. TokenVerifier

Utilisé par les passerelles API / services pour valider localement les jetons.

public interface TokenVerifier {
    <T> VerifiedData<T> verify(String token, Function<String, T> deserializer)
        throws BrokerExtractionException;
}

3. TokenRevoker

Utilisé par les consoles d'administration ou modules de sécurité pour révoquer des sessions.

public interface TokenRevoker {
    void revoke(Scope scope, String sequenceId);
}

4. TokenTracker

Utilisé pour inspecter et lister les informations sur les sessions actives ou révoquées.

public interface TokenTracker {
    List<SessionInfo> listActive(Scope scope);
}

Clés de Configuration

Configurez les instances Veridot au démarrage à l'aide des variables d'environnement (ou propriétés système) suivantes :

Variable d'environnement Valeur par défaut Description
VDOT_KEYS_ROTATION_MINUTES 60 Intervalle de rotation des clés de signature éphémères.
VDOT_RECONCILIATION_INTERVAL_MINUTES 10 Fréquence de la boucle de réconciliation de filigrane (watermark).
VDOT_CAPABILITY_CACHE_TTL_SECONDS 60 Durée de mise en cache positive des capacités d'autorisation.
VDOT_CAPABILITY_NEGATIVE_CACHE_TTL_SECONDS 5 Durée de mise en cache négative (échec d'autorisation) des capacités.
