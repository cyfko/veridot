API Reference

This reference details Veridot's core Java interfaces, constructors, methods, and exceptions.

Core Interfaces

1. DataSigner

The interface used to issue and sign new user session tokens.

public interface DataSigner {
    /**
     * Generates a signed cryptographic token for the given payload.
     *
     * @param data The payload object to serialize and sign.
     * @param configurer The configuration options (validity, sequenceId, etc.).
     * @return The signed token.
     * @throws DataSerializationException If serialization fails.
     */
    String sign(Object data, Configurer configurer) throws DataSerializationException;
}

2. TokenVerifier

The interface used to validate and extract payloads from tokens locally.

public interface TokenVerifier {
    /**
     * Validates the signature, liveness, and configuration of the token.
     *
     * @param token The token string.
     * @param deserializer Function to deserialize the JSON payload to type T.
     * @return A VerifiedData container containing the payload and validity info.
     * @throws BrokerExtractionException If the metadata cannot be resolved or is stale.
     */
    <T> VerifiedData<T> verify(String token, java.util.function.Function<String, T> deserializer)
        throws BrokerExtractionException;
}

3. TokenRevoker

The interface used to revoke active sessions cryptographically.

public interface TokenRevoker {
    /**
     * Revokes a specific session sequence or all sessions within a group.
     *
     * @param scope The scope of the target (e.g. Scope.group("user-99")).
     * @param sequenceId The unique session key to revoke, or null to revoke the entire group.
     */
    void revoke(Scope scope, String sequenceId);
}

Constructors & Methods

GenericSignerVerifier

The standard concrete implementation of DataSigner, TokenVerifier, and TokenRevoker.

  • public GenericSignerVerifier(Broker broker, TrustRoot trustRoot)
    Creates a verifier-only instance. Cannot be used to sign or revoke.
  • public GenericSignerVerifier(Broker broker, TrustRoot trustRoot, String issuerId, java.security.PrivateKey privateKey, byte sigAlg)
    Creates an issuance and verification instance.
  • public void close()
    Closes background threads, database connection pools, or Kafka consumers.

Exceptions

Exception Class Parent Class Description
BrokerExtractionException RuntimeException Thrown when verifier fails to extract metadata from the broker or database.
DataSerializationException Exception Thrown when token payload fails JSON serialization.
EnvelopeValidationException Exception Thrown when a binary envelope violates wire formatting or checks.
RuleValidationException Exception Thrown when a claim-based custom validation rule fails.
