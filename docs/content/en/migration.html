Migration Guide: V3 to V4

This guide explains how to migrate your existing systems from Veridot Protocol V3 to Protocol V4.

V3 vs V4 Differences

Veridot V4 introduces major changes to the wire format, authorization structure, and key management:

  • Wire Format: The envelope magic bytes changed from 0x56 0x33 to 0x56 0x44, and version marker field validation is strictly enforced.
  • Capability Model: V4 introduces structural authorization via the CAPABILITY entry type. In V3, authorization was handled implicitly or by application code. In V4, any config or fence write must prove its capability chain.
  • Fencing and evictions: V4 introduces monotonic FENCE tokens to securely manage session quotas under concurrent signers.

Migrating Key Material

In Protocol V3, public keys were stored as raw certificate strings or Jwks. In V4, keys are distributed using KEY_EPOCH payloads carrying DER-encoded public key bytes.

To upgrade your keys:

  1. Export the long-term public keys of your issuers as PEM/DER files.
  2. Publish a CAPABILITY grant for each issuer, authorizing them to publish on their respective scopes.
  3. Publish the initial KEY_EPOCH containing the ephemeral signing key under the newly authorized issuer.

Coexistence & Zero-Downtime Upgrades

During the upgrade window, gateways must verify both V3 and V4 tokens. To implement a zero-downtime migration, configure a dual-verification pipeline:

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.exceptions.*;

public class DualVerifier {
    private final TokenVerifier v4Verifier;
    private final io.github.cyfko.veridot.v3.TokenVerifier v3Verifier; // From legacy v3 library

    public DualVerifier(TokenVerifier v4Verifier, io.github.cyfko.veridot.v3.TokenVerifier v3Verifier) {
        this.v4Verifier = v4Verifier;
        this.v3Verifier = v3Verifier;
    }

    public VerifiedData verify(String token) throws Exception {
        // Step 1: Detect version from token header or envelope bytes
        if (token.startsWith("VD04") || token.startsWith("{\"alg\":") || token.startsWith("eyJ")) {
            try {
                // Try V4 verification path
                return v4Verifier.verify(token, payload -> payload);
            } catch (Exception e) {
                // If it fails, check if it's a legacy V3 token
                if (isV3Token(token)) {
                    return v3Verifier.verify(token);
                }
                throw e;
            }
        }
        
        // Fallback to V3
        return v3Verifier.verify(token);
    }

    private boolean isV3Token(String token) {
        // Custom logic to identify V3 signature markers
        return token.contains("v3-sig-marker");
    }
}
