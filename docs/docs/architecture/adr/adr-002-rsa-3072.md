# ADR 002: Algorithm Selection & Post-Quantum Readiness

## Context
Signature verification speed and security are paramount. Traditional RSA and ECDSA have timing channel risks or nonce vulnerabilities. Moreover, the protocol must be ready for Post-Quantum Cryptography (PQC).

## Decision
- **Ed25519** and **RSA_PSS_SHA256** are the default baseline algorithms.
- **ML_DSA_65** (FIPS 204) is supported to provide post-quantum readiness.
- ECDSA is supported but discouraged due to strict deterministic nonce requirements.
- The `DIRECT` mode (JWT) does not support ML_DSA_65 due to lack of JWT header standards; ML_DSA_65 is used in `NATIVE` and `PRIVATE` modes.

## Consequences
- Veridot V5 is post-quantum ready.
- Verification remains sub-millisecond, especially when Ed25519 is used.
