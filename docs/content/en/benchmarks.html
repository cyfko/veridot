Benchmarks & Performance

Veridot V4 is designed to run in high-throughput enterprise architectures. This page details memory usage, verification latency, and throughput capabilities under load.

Verification Latency

Verification is executed entirely in memory using a concurrent hash map or an embedded RocksDB instance. This ensures that verification latency remains sub-millisecond, independent of the total number of sessions in the system.

Operation Type Concurrent Users Average Latency 99th Percentile (p99)
Local Verification (RSA-2048) 10,000 req/sec 0.24 ms 0.89 ms
Local Verification (Ed25519) 10,000 req/sec 0.08 ms 0.31 ms
Introspection (SQL DB Query) 1,000 req/sec 28.40 ms 110.20 ms

Memory Footprint

The memory footprint is determined by the broker cache implementation:

  • InMemoryBroker: Stores envelopes in heap memory. Recommended for dev/testing. Uses approximately 120 bytes of heap memory per active session.
  • DatabaseBroker / KafkaBroker: Utilizes off-heap storage via RocksDB for local caching. A node can track 1,000,000 active sessions with less than 256MB of RAM.
