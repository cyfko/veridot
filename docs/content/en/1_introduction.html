Introduction to Veridot

Veridot is a high-performance cryptographic token verification protocol and Java library designed for modern, large-scale distributed architectures. It addresses the classic trade-off between instant revocation verification and system scalability.

The Distributed Revocation Challenge

In distributed systems, validating identity and authorization tokens typically falls into one of two patterns, both of which introduce operational pain points:

  • Stateful Token Introspection (Slow & Centralized): The verifying service makes a network call (HTTP/RPC or direct database lookup) to verify if a token is still active. This creates a critical bottleneck, introduces latency, and forms a single point of failure (SPOF).
  • Stateless JWT Validation (Fast but Hard to Revoke): Verifying services cryptographically validate the JWT locally using a public key. While extremely fast (sub-millisecond), revoking a compromised token instantly is practically impossible without resorting to blacklists that reintroduce state bottlenecks.

The Veridot Solution

Veridot achieves the speed of local stateless verification with the instant responsiveness of a stateful revocation system. It is built on three core pillars:

Core Pillars
  • Localized Verification: Verifying nodes cache security contexts and validate tokens locally with cryptographic guarantees, achieving sub-millisecond latencies.
  • Asynchronous Status Replication: State updates, including configurations, key epochs, and liveness signals, are replicated asynchronously to all nodes via a fast shared broker (e.g., Kafka or SQL databases).
  • Version Watermark Invariants: A strict anti-rollback system ensures that verifying nodes can never be tricked by stale state replays, enforcing absolute safety.
