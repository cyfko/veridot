# DataSigner & BasicConfigurer

The `DataSigner` interface is responsible for generating signed tokens (JWTs) or lightweight identifiers (`messageId`) and publishing the corresponding verification metadata envelopes to the Broker.

---

## 1. Interface: DataSigner

### Method
```java
String sign(Object data, Configurer configurer) 
    throws DataSerializationException, BrokerTransportException;
```

- **Role**: Issues a signed session token containing a payload.
- **Parameters**:
  - `data`: The payload object to embed (must not be `null`).
  - `configurer`: The signing request configuration (must not be `null`).
- **Returns**:
  - In `DIRECT` mode: The complete cryptographically signed token string (JWT).
  - In `INDIRECT` mode: The lightweight `messageId` (e.g. `4:groupId:sequenceId`). The token itself is stored in the broker.
- **Exceptions**:
  - `DataSerializationException`: Thrown if the payload cannot be serialized to a string.
  - `BrokerTransportException`: Thrown if metadata publication to the Broker fails.
  - `IllegalArgumentException`: Thrown if configuration parameters are invalid (e.g., negative duration).

---

## 2. Configuration: DataSigner.Configurer

Defines the parameters for a token issuance request:

- **`getGroupId()`**: Returns the business identity string (1–125 characters, no whitespaces or `:`). **Precondition**: Must not be null or blank.
- **`getSequenceId()`**: Optional sequence key. If null, a random UUID is auto-generated by the signer.
- **`getDistribution()`**: Returns `DistributionMode.DIRECT` or `DistributionMode.INDIRECT`.
- **`getDuration()`**: Validity window in seconds. **Precondition**: Must be greater than 0.
- **`getSerializer()`**: Converter function transforming payload object to a string.

---

## 3. Concrete Builder: BasicConfigurer

`BasicConfigurer` is the standard implementation of `DataSigner.Configurer` providing a fluent builder pattern.

### Preconditions & Invariants
- **`groupId`**: Mandatory. Cannot contain colons (`:`), commas (`,`), pipes (`|`), or control characters.
- **`validity`**: Mandatory. Must be a positive long integer representing duration in seconds.
- **`serializer`**: Defaults to Jackson's `ObjectMapper` unless overridden via `serializedBy`.

---

## 4. Code Examples

### Standard Java SE (JSON POJO Serializer)
```java
import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.BasicConfigurer;

public class SignerExample {
    
    public record UserClaims(String email, String role, String tenantId) {}

    public String issueUserSession(DataSigner signer, UserClaims claims) {
        // Build the signing request
        BasicConfigurer config = BasicConfigurer.builder()
                .groupId(claims.tenantId())        // e.g. "tenant-A"
                .sequenceId("session-" + claims.email()) // Specific session ID
                .distribution(DistributionMode.DIRECT)  // Return JWT directly
                .validity(3600)                     // 1 hour validity
                .build();
        
        // Sign and publish metadata
        return signer.sign(claims, config);
    }
}
```

### Indirect Mode (Large Payloads)
In `INDIRECT` mode, the payload is stored directly in the Broker, and the signer only returns a lightweight `messageId`. Use this mode when sending large claims lists across network links:

```java
String messageId = signer.sign(largePayload,
    BasicConfigurer.builder()
        .groupId("analytics-group")
        .distribution(DistributionMode.INDIRECT)
        .validity(1800) // 30 minutes
        .build()
);
// Returns: "4:analytics-group:7b8d4f..." (Broker key)
```
