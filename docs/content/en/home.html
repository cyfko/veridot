Introduction to Veridot

Veridot is an enterprise-grade, high-performance cryptographic token verification protocol and Java library designed for large-scale distributed architectures. It bridges the gap between fast, stateless local JWT verification and instant, stateful token revocation.

Key Highlights

Why choose Veridot V4?
  • ✓ Sub-millisecond Latency: Cryptographic token validation is executed entirely in memory using local thread-safe caches, avoiding synchronous database calls.
  • ✓ Decentralized Revocation: Revocations propagate asynchronously to all microservices via a replication broker in less than a second.
  • ✓ Rollback-Proof Monotonicity: Veridot V4 enforces version watermarks on all state messages, preventing replay attacks and out-of-order state updates.
  • ✓ Production-Ready: Native integrations for Spring Boot, Apache Kafka, and relational databases.

The Authentication Trade-Off

Modern microservice architectures require fast, secure, and highly scalable user authentication. In typical setups, developers choose between two flawed architectural patterns:

Operational Metric Traditional Stateless JWT OAuth Introspection (Stateful) Veridot V4
Verification Latency < 1 ms (Local cryptographic check) 15 ms - 150 ms (Synchronous network call) < 1 ms (Local cached check)
Revocation Propagation Not possible (Must wait for TTL to expire) Instantaneous < 1 second (Asynchronous event propagation)
Database Load Zero (No database lookups) High (Queries database for every check) Extremely Low (Only reads updates from broker)
Network Failure Resilience Excellent (Independent verification) Very Poor (Authentication fails if server is down) High (Falls back to local cache & watermark)

Minimal Usage Example

The following example initializes the Veridot verification service, generates a session token for a user, verifies it locally, and revokes it:

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import java.security.KeyPairGenerator;
import java.security.KeyPair;
import java.security.PublicKey;

public class MinimalExample {
    public static void main(String[] args) throws Exception {
        // Initialize simple in-memory broker
        Broker broker = new InMemoryBroker();
        
        // Generate key pair
        KeyPair keyPair = KeyPairGenerator.getInstance("RSA").generateKeyPair();
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override public PublicKey resolve(String issuer) { return keyPair.getPublic(); }
            @Override public boolean isRootIdentity(String issuer) { return "auth-issuer".equals(issuer); }
        };

        // Initialize verifier
        GenericSignerVerifier veridot = new GenericSignerVerifier(
            broker, trustRoot, "auth-issuer", keyPair.getPrivate(), (byte) 0x01
        );

        // Sign token
        Configurer configurer = BasicConfigurer.builder()
            .groupId("user-123")
            .sequenceId("session-xyz")
            .validity(3600)
            .build();
        String token = veridot.sign("{\"user\":\"Alice\"}", configurer);

        // Verify locally
        VerifiedData verified = veridot.verify(token, payload -> payload);
        System.out.println("Is session valid? " + verified.isLive());

        // Revoke instantly
        veridot.revoke(Scope.group("user-123"), "session-xyz");
        veridot.close();
    }
}
