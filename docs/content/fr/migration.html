Guide de Migration : V3 vers V4

Ce guide détaille les étapes nécessaires pour migrer votre infrastructure de Veridot Protocol V3 vers la version V4.

Différences Majeures entre V3 et V4

Veridot V4 introduit des changements structuraux majeurs :

  • Format d'Enveloppe : Les octets magiques (magic bytes) passent de 0x56 0x33 à 0x56 0x44.
  • Modèle d'Autorisation (Capabilities) : V4 introduit le type d'entrée CAPABILITY. Toute modification de configuration ou d'exclusion de quota doit être signée par un émetteur dont la chaîne d'autorisation remonte au TrustRoot.
  • Fencing : La gestion des quotas de sessions concurrentes repose sur le type d'entrée FENCE et son compteur monotone pour éviter les conflits d'écriture.

Migration des Clés

Dans la version V3, les clés étaient déclarées dans des fichiers de configuration ou via Jwks. En V4, elles sont distribuées par le broker via les entrées KEY_EPOCH contenant la clé publique au format DER.

Pour migrer vos clés :

  1. Exportez les clés publiques long-terme de vos émetteurs au format DER.
  2. Publiez une entrée CAPABILITY autorisant chaque identité d'émetteur sur ses scopes correspondants.
  3. Émettez la première entrée KEY_EPOCH contenant la clé de signature éphémère.

Coexistence et Transition Transparente

Pendant la phase de transition, les passerelles doivent valider à la fois les jetons V3 et V4. Voici comment configurer un pipeline de double vérification :

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.exceptions.*;

public class DualVerifier {
    private final TokenVerifier v4Verifier;
    private final io.github.cyfko.veridot.v3.TokenVerifier v3Verifier; // Issue de la bibliothèque V3 héritée

    public DualVerifier(TokenVerifier v4Verifier, io.github.cyfko.veridot.v3.TokenVerifier v3Verifier) {
        this.v4Verifier = v4Verifier;
        this.v3Verifier = v3Verifier;
    }

    public VerifiedData verify(String token) throws Exception {
        // Étape 1 : Détecter la version du jeton
        if (token.startsWith("VD04") || token.startsWith("{\"alg\":") || token.startsWith("eyJ")) {
            try {
                // Tentative avec le validateur V4
                return v4Verifier.verify(token, payload -> payload);
            } catch (Exception e) {
                // En cas d'échec, vérifier s'il s'agit d'un jeton V3 hérité
                if (isV3Token(token)) {
                    return v3Verifier.verify(token);
                }
                throw e;
            }
        }
        
        // Validation par défaut en V3
        return v3Verifier.verify(token);
    }

    private boolean isV3Token(String token) {
        // Logique personnalisée d'identification des jetons V3
        return token.contains("v3-sig-marker");
    }
}
