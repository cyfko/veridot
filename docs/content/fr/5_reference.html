Référence API & Configuration

Ce guide référence de manière exhaustive les interfaces Java majeures de Veridot, les exceptions levées et les clés de configuration système.

Surfaces des API Publiques Java

1. DataSigner

L'interface utilisée pour émettre et signer les jetons de session.

public interface DataSigner {
    /**
     * Génère un jeton cryptographique signé pour la payload fournie.
     *
     * @param data L'objet de données à sérialiser et signer.
     * @param configurer Les options de configuration (validité, sequenceId, mode de distribution).
     * @return Le jeton JWT signé (mode DIRECT) ou l'identifiant opaque (mode INDIRECT).
     * @throws DataSerializationException Si la payload ne peut pas être sérialisée.
     */
    String sign(Object data, Configurer configurer) throws DataSerializationException;
}

2. TokenVerifier

L'interface utilisée pour valider et extraire les payloads localement.

public interface TokenVerifier {
    /**
     * Valide la signature, le signal de vie (liveness) et la configuration du jeton.
     *
     * @param token Le jeton JWT ou l'ID de message à vérifier.
     * @param deserializer Fonction pour désérialiser la payload JSON en type T.
     * @return Un conteneur VerifiedData contenant la payload et les infos de validité.
     * @throws BrokerExtractionException Si les métadonnées ne peuvent pas être lues ou sont expirées.
     */
    <T> VerifiedData<T> verify(String token, Function<String, T> deserializer)
        throws BrokerExtractionException;
}

3. TokenRevoker

L'interface utilisée pour révoquer cryptographiquement les sessions actives.

public interface TokenRevoker {
    /**
     * Révoque une session spécifique ou toutes les sessions d'un groupe.
     *
     * @param scope Le scope de la cible (ex. Scope.group("user-99")).
     * @param sequenceId L'identifiant unique de la session à révoquer, ou null pour révoquer tout le groupe.
     */
    void revoke(Scope scope, String sequenceId);
}

Propriétés de Configuration Détaillées

Configurez le moteur Veridot au démarrage à l'aide des variables d'environnement suivantes :

Propriété Système Variable d'Env Valeur par défaut Type Description
veridot.keys.rotation.minutes VDOT_KEYS_ROTATION_MINUTES 60 entier Fréquence de rotation en arrière-plan des clés de signature éphémères pour limiter l'exposition des clés.
veridot.reconciliation.interval.minutes VDOT_RECONCILIATION_INTERVAL_MINUTES 10 entier Intervalle de temps entre les réconciliations périodiques de filigrane (watermark) avec le broker.
veridot.capability.cache.ttl.seconds VDOT_CAPABILITY_CACHE_TTL_SECONDS 60 entier Durée de conservation en cache des droits d'autorisation et configurations de confiance d'un émetteur.
veridot.capability.negative.cache.ttl.seconds VDOT_CAPABILITY_NEGATIVE_CACHE_TTL_SECONDS 5 entier Durée de conservation des échecs de validation d'autorisations. Protège le broker contre les attaques par force brute.
veridot.max.sessions.default VDOT_MAX_SESSIONS_DEFAULT -1 entier Limite par défaut du nombre maximal de sessions actives concurrentes par groupe. -1 signifie illimité.
