Development Roadmap

This roadmap outlines the planned features and future directions for the Veridot protocol and library.

Short-Term (V4.x Revisions)

  • SIMD-Accelerated Cryptography: Utilize Java's Vector API (Project Panama) to achieve SIMD-accelerated signature validation for RSA and Ed25519.
  • Native Mobile SDKs: Release lightweight, native C-based core libraries with bindings for Android (JNI) and iOS (Swift/C++).

Medium-Term (V5.0 Planning)

  • WebAssembly (Wasm) Port: Compile the Veridot verification engine to WebAssembly. This will allow secure, high-performance token verification at the CDN Edge (Cloudflare Workers, Fastly Compute@Edge) and in web browsers.
  • Zero-Knowledge Revocation Proofs: Explore ZK-proof cryptography to verify token liveness without revealing user identities or session IDs to the database/broker.

Long-Term (Research & Development)

  • AI-Driven Validation Rules: Build an optional, intelligent rule validation engine that learns normal request profiles and flags anomalous token usage (e.g. impossible travel time) locally at the gateway.
