Tutoriels

Ces tutoriels pratiques vous guident pas-à-pas pour intégrer Veridot, valider des jetons au niveau passerelle et écrire des règles de validation personnalisées.

Tutoriel 1 : Construire votre Premier Projet

Nous allons mettre en place un projet Java standard avec Maven et implémenter une logique d'authentification simple.

Étape A : Structure du Projet

my-veridot-project/
├── pom.xml
└── src/
    └── main/
        └── java/
            └── com/
                └── example/
                    └── AuthServer.java

Étape B : Fichier pom.xml

<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-veridot-project</artifactId>
    <version>1.0-SNAPSHOT</version>
    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
    </properties>
    <dependencies>
        <dependency>
            <groupId>io.github.cyfko</groupId>
            <artifactId>veridot-core</artifactId>
            <version>4.0.0</version>
        </dependency>
    </dependencies>
</project>

Tutoriel 2 : Valider un Jeton

Ce tutoriel montre comment une passerelle API (Gateway) vérifie un jeton utilisateur. Veridot valide la structure, la signature et l'état actif (liveness) en local.

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import io.github.cyfko.veridot.core.exceptions.*;
import java.security.PublicKey;

public class GatewayVerifier {
    private final TokenVerifier verifier;

    public GatewayVerifier(Broker broker, PublicKey authorityPublicKey) {
        // Résolution de clé et vérification de l'autorité racine
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override
            public PublicKey resolve(String issuer) {
                return authorityPublicKey;
            }
            @Override
            public boolean isRootIdentity(String issuer) {
                return "auth-authority-01".equals(issuer);
            }
        };

        // Initialisation du TokenVerifier
        this.verifier = new GenericSignerVerifier(broker, trustRoot);
    }

    public String handleRequest(String authHeader) {
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            return "401 Unauthorized - Jeton absent";
        }
        
        String token = authHeader.substring(7);
        try {
            // Validation locale
            VerifiedData verified = verifier.verify(token, payload -> payload);
            
            // Succès
            return "200 OK - Accès autorisé. Profil : " + verified.data();
        } catch (BrokerExtractionException e) {
            // Révocation détectée
            return "403 Forbidden - Jeton invalide ou révoqué : " + e.getMessage();
        } catch (Exception e) {
            // Échec cryptographique
            return "401 Unauthorized - Échec de validation : " + e.getMessage();
        }
    }
}

Tutoriel 3 : Écrire une Règle Personnalisée

Les règles s'exécutent lors de la validation du jeton. Vous pouvez restreindre l'accès en fonction des claims du jeton en implémentant l'interface Rule.

L'exemple ci-dessous vérifie que l'adresse e-mail de l'utilisateur a bien été validée :

package com.example;

import io.github.cyfko.veridot.core.Rule;
import io.github.cyfko.veridot.core.exceptions.RuleValidationException;
import java.util.Map;

public class EmailVerificationRule implements Rule {
    
    @Override
    public String getName() {
        return "EMAIL_VERIFICATION_RULE";
    }

    @Override
    public void validate(Map claims) throws RuleValidationException {
        Object emailVerified = claims.get("email_verified");
        
        if (emailVerified == null) {
            throw new RuleValidationException("Le statut de validation d'e-mail est absent du jeton");
        }
        
        if (emailVerified instanceof Boolean && !((Boolean) emailVerified)) {
            throw new RuleValidationException("L'adresse e-mail doit être validée");
        }
        
        if (emailVerified instanceof String && !"true".equalsIgnoreCase((String) emailVerified)) {
            throw new RuleValidationException("L'adresse e-mail doit être validée");
        }
    }
}

Pour enregistrer votre règle personnalisée, passez-la au builder lors de l'initialisation :

GenericSignerVerifier verifier = GenericSignerVerifier.builder(broker, trustRoot)
    .addRule(new EmailVerificationRule())
    .build();
