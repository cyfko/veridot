Guide de Démarrage

Ce guide vous accompagne dans l'installation de Veridot Core, la configuration d'un environnement de base et la création d'un premier programme complet de signature et vérification de jetons.

Prérequis

  • Java Development Kit (JDK) 17 ou supérieur (JDK 21 recommandé)
  • Apache Maven 3.8+ ou Gradle 7.4+
  • Une source de temps synchronisée de manière sécurisée (NTP)

Installation

Ajoutez la dépendance Veridot Core à votre projet. L'édition communautaire (CE) fournit les brokers en mémoire, par fichier, et base de données relationnelle. Le broker Kafka nécessite le module veridot-kafka.

<dependency>
    <groupId>io.github.cyfko</groupId>
    <artifactId>veridot-core</artifactId>
    <version>4.0.0</version>
</dependency>
implementation 'io.github.cyfko:veridot-core:4.0.0'

Configuration Minimale

Veridot se configure à l'aide de propriétés système ou de variables d'environnement. La configuration minimale définit l'intervalle de rotation des clés et le TTL de cache :

# Configuration par variables d'environnement
export VDOT_KEYS_ROTATION_MINUTES=60
export VDOT_RECONCILIATION_INTERVAL_MINUTES=10
export VDOT_CAPABILITY_CACHE_TTL_SECONDS=60

Exemple Hello World

Le code suivant illustre un cycle complet d'émission, de vérification locale et de révocation immédiate avec signature RSA :

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import io.github.cyfko.veridot.core.exceptions.*;
import java.security.*;

public class HelloWorld {
    public static void main(String[] args) throws Exception {
        // 1. Initialisation du broker
        Broker broker = new InMemoryBroker();

        // 2. Génération d'une paire de clés RSA 2048-bit
        KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
        keyGen.initialize(2048);
        KeyPair pair = keyGen.generateKeyPair();

        // 3. Définition du TrustRoot
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override
            public PublicKey resolve(String issuer) {
                return pair.getPublic();
            }
            @Override
            public boolean isRootIdentity(String issuer) {
                return "root-authority".equals(issuer);
            }
        };

        // 4. Initialisation du moteur Veridot
        GenericSignerVerifier engine = new GenericSignerVerifier(
            broker,
            trustRoot,
            "root-authority",
            pair.getPrivate(),
            (byte) 0x01 // RSA-SHA256
        );

        // 5. Signature d'un jeton de session pour le groupe "users"
        Configurer config = BasicConfigurer.builder()
            .groupId("users")
            .sequenceId("sess-01")
            .validity(300) // 5 minutes
            .build();
        
        String token = engine.sign("{\"userId\":\"1001\",\"role\":\"editor\"}", config);
        System.out.println("Jeton généré : " + token);

        // 6. Vérification du jeton
        try {
            VerifiedData verified = engine.verify(token, payload -> payload);
            System.out.println("Résultat de validation : SUCCÈS");
            System.out.println("Payload : " + verified.data());
        } catch (Exception e) {
            System.out.println("Résultat de validation : ÉCHEC - " + e.getMessage());
        }

        // 7. Révocation de la session sess-01
        System.out.println("Révocation de la session sess-01...");
        engine.revoke(Scope.group("users"), "sess-01");

        // 8. Nouvelle vérification du jeton révoqué (doit échouer)
        try {
            engine.verify(token, payload -> payload);
            System.out.println("Résultat de validation : SUCCÈS (Erreur de comportement !)");
        } catch (BrokerExtractionException e) {
            System.out.println("Résultat de validation : ÉCHEC (Comportement correct !) : " + e.getMessage());
        }

        engine.close();
    }
}
