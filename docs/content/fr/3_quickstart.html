Guide de Démarrage Rapide

Démarrez avec Veridot en moins de 5 minutes. Apprenez à intégrer la dépendance et à exécuter votre première signature et validation cryptographique.

Installation

Sélectionnez votre outil de build pour ajouter la dépendance Veridot Core à votre projet :

<dependency>
    <groupId>io.github.cyfko</groupId>
    <artifactId>veridot-core</artifactId>
    <version>4.0.0</version>
</dependency>
implementation 'io.github.cyfko:veridot-core:4.0.0'

Première Signature & Vérification

Voici un exemple complet en Java démontrant comment initialiser un signataire/vérificateur, signer une payload utilisateur et la valider localement :

import io.github.cyfko.veridot.core.*;
import io.github.cyfko.veridot.core.impl.*;
import java.security.*;

public class VeridotQuickstart {
    public static void main(String[] args) throws Exception {
        // 1. Initialise un broker simple en mémoire (pour les tests)
        Broker broker = new InMemoryBroker();
        
        // 2. Génère des clés cryptographiques à long terme (RSA 2048)
        KeyPairGenerator kpg = KeyPairGenerator.getInstance("RSA");
        kpg.initialize(2048);
        KeyPair keyPair = kpg.generateKeyPair();
        
        // 3. Définit l'ancre de confiance TrustRoot
        TrustRoot trustRoot = new PublicKeyTrustRoot() {
            @Override
            public PublicKey resolve(String issuer) {
                return keyPair.getPublic();
            }
            @Override
            public boolean isRootIdentity(String issuer) {
                return "root-admin".equals(issuer);
            }
        };

        // 4. Instancie le service GenericSignerVerifier
        GenericSignerVerifier service = new GenericSignerVerifier(
            broker,
            trustRoot,
            "root-admin",
            keyPair.getPrivate(),
            (byte) 0x01 // Algorithme de signature d'enveloppe (SHA256withRSA)
        );

        // 5. Configure l'opération de signature (validité de session)
        Configurer configurer = BasicConfigurer.builder()
            .groupId("user-sessions")
            .sequenceId("session-xyz-123")
            .validity(300) // valide pendant 5 minutes
            .build();

        // 6. Signe la payload utilisateur
        String token = service.sign("User ID: 9942", configurer);
        System.out.println("Token JWT ou ID de Message généré : " + token);

        // 7. Vérifie le jeton localement
        VerifiedData<String> verified = service.verify(token, payload -> payload);
        System.out.println("Vérification Réussie ! Données : " + verified.data());
        System.out.println("Statut Session : " + (verified.isLive() ? "ACTIF" : "EXPIRE"));
        
        // Arrête les planificateurs internes
        service.close();
    }
}
Modes de Distribution

Par défaut, DistributionMode.DIRECT renvoie un JWT autonome. Vous pouvez utiliser .distribution(DistributionMode.INDIRECT) pour renvoyer un identifiant opaque court (pratique pour minimiser l'overhead réseau).
